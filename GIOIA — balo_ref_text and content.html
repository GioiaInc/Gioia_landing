What We're Building
Messaging that feels like talking

Most of what we communicate isn't in the words themselves. It's in the pauses, the speed, the tone, the timing. Current messaging platforms throw all of that away. You get a flat string of text with no context for how the other person actually feels.

balo is a messaging app built around ambient AI. It runs lightweight sentiment analysis on-device using micro transformer models, then reflects the emotional state of a conversation through the interface ‚Äî colors shift, visual atmospheres change, and gentle nudges appear when the tone gets heated. The AI processing happens locally on the phone. Mood scores (not messages) are synced between users via WebSocket.

GIOIA is the company behind balo. Over time, the behavioral and emotional patterns that balo generates (stored as anonymized vector embeddings, not raw messages) have applications beyond messaging ‚Äî in marketing, health, education, and AI training. But the product comes first.

Technology
Built on-device, designed to scale
The AI inference runs on-device. Server infrastructure handles message delivery and score syncing. Compositional AI features (writing assistance, smart reminders) come later, when there's revenue to support inference costs.
üß†

On-Device Micro Transformers

Sentiment analysis runs locally using small transformer models. Mood scores (not messages) are exchanged between users via WebSocket. This keeps AI inference costs near zero and means message content doesn't need to be processed on a server.

üë§

Digital Twins

Over time, each user builds a vector-based profile of their communication style. These aren't fine-tuned models ‚Äî they're continuously updated embeddings retrieved in-context as needed. Whether processing happens on-device or server-side is still being explored.

Data Architecture

User data is stored as vector embeddings, not baked into models. This avoids the diminishing returns of fine-tuning on user data and keeps things flexible. No custom model training ‚Äî frontier labs are moving too fast for that to yield sustainable ROI at this stage.

Data Sovereignty

Most global platforms extract user data from emerging markets and process it in foreign jurisdictions. GIOIA is building differently ‚Äî data sovereignty is a core architectural decision, not a compliance afterthought.

The biggest barrier to cross-platform data use isn't technical ‚Äî it's legal. Incompatible privacy policies and regional laws create friction that a locally-operated company can navigate more effectively than a global platform can.
